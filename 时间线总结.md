# 昨天版本（39f095c）的完整时间线和时间段总结

## 核心状态枚举

### BicycleStatus（自行车状态）
```python
# 待审核状态
PENDING_APPROVAL = "PENDING_APPROVAL"  # 待管理员审核

# 库存状态
IN_STOCK = "IN_STOCK"  # 在库存中，可被预约

# 卖家流程状态（卖家卖车）
PENDING_SELLER_SLOT_SELECTION = "PENDING_SELLER_SLOT_SELECTION"  # 管理员已提出时间段，等待卖家选择
PENDING_ADMIN_CONFIRMATION_SELLER = "PENDING_ADMIN_CONFIRMATION_SELLER"  # 卖家已选择时间段，等待管理员确认
RESERVED = "RESERVED"  # 管理员已确认，等待线下交车（卖家送车到指定地点）

# 买家流程状态（买家买车）
PENDING_BUYER_SLOT_SELECTION = "PENDING_BUYER_SLOT_SELECTION"  # 管理员已提出时间段，等待买家选择
PENDING_ADMIN_CONFIRMATION_BUYER = "PENDING_ADMIN_CONFIRMATION_BUYER"  # 买家已选择时间段，等待管理员确认
PENDING_PICKUP = "PENDING_PICKUP"  # 管理员已确认时间段，等待买家线下提车
SOLD = "SOLD"  # 交易完成
```

### AppointmentStatus（预约状态）
```python
PENDING = "PENDING"      # 待确认
CONFIRMED = "CONFIRMED"  # 已确认
COMPLETED = "COMPLETED"  # 已完成
CANCELLED = "CANCELLED"  # 已取消
```

### TimeSlot（时间段）
```python
appointment_type: 时间段类型
- "drop-off": 管理员送车/买家取车（适用于买家流程）
- "pick-up": 管理员取车/卖家送车（适用于卖家流程）

is_booked: 是否已被预订
- "true": 已被用户选择
- "false": 未被选择
```

## 完整时间线流程

### 流程一：卖家流程（卖家卖车给平台）

#### 步骤 1：卖家登记车辆
- **前端操作**：卖家在"登记车辆"页面提交车辆信息
- **后端 API**：`POST /bicycles/`
- **状态变化**：
  - 自行车状态：`PENDING_APPROVAL`
  - 预约：无

#### 步骤 2：管理员审核通过
- **前端操作**：管理员在管理后台点击"审核通过"
- **后端 API**：`PUT /bicycles/{bike_id}/approve`
- **状态变化**：
  - 自行车状态：`PENDING_APPROVAL` → `IN_STOCK`
  - 预约：无

#### 步骤 3：管理员提出时间段
- **前端操作**：管理员点击"提出时间段"，输入多个可选时间段
- **后端 API**：`POST /bicycles/{bike_id}/propose-slots`
- **后端逻辑**：
  - 创建预约（Appointment），类型为 `pick-up`（买家取车类型，但这里是卖家流程）
  - 创建时间段（TimeSlot），`appointment_type = "pick-up"`（管理员取车）
  - 发送私信通知卖家
- **状态变化**：
  - 自行车状态：`IN_STOCK` → `PENDING_SELLER_SLOT_SELECTION`
  - 预约状态：`PENDING`
  - 时间段：`is_booked = "false"`（多个时间段）

#### 步骤 4：卖家选择时间段
- **前端操作**：卖家在"我的时间段"页面选择一个时间段
- **后端 API**：`POST /time_slots/select`
- **后端逻辑**：
  - 验证时间段类型是否匹配（卖家需要选择 `appointment_type = "pick-up"` 的时间段）
  - 标记时间段为已预订
  - 更新预约的 `time_slot_id`
  - 发送私信通知管理员确认
- **状态变化**：
  - 自行车状态：`PENDING_SELLER_SLOT_SELECTION` → `PENDING_ADMIN_CONFIRMATION_SELLER`
  - 预约状态：`PENDING`（保持不变）
  - 时间段：`is_booked = "true"`

#### 步骤 5：管理员确认时间段（⏳ 等待确认的自行车）
- **前端操作**：管理员在管理后台看到"⏳ 等待确认的自行车（卖家已选时间段）"，点击"确认交易"
- **后端 API**：`POST /bicycles/{bike_id}/confirm`
- **后端逻辑**：
  - 检查预约是否有时间段
  - 根据预约类型设置自行车状态：
    - `appointment.type == "drop-off"` → `RESERVED`（等待线下交车）
    - `appointment.type == "pick-up"` → `SOLD`（交易完成）
  - 更新预约状态为 `CONFIRMED`
  - 发送私信通知卖家
- **状态变化**：
  - 自行车状态：`PENDING_ADMIN_CONFIRMATION_SELLER` → `RESERVED` 或 `SOLD`
  - 预约状态：`PENDING` → `CONFIRMED`

#### 步骤 6：线下交易完成（卖家送车）
- **场景**：卖家按约定时间将车送到指定地点，管理员接收车辆
- **后续操作**：
  - 如果自行车状态是 `RESERVED`：管理员可以再确认交易完成，改为 `IN_STOCK` 或 `SOLD`

## 时间段类型对应关系

| 流程类型 | 预约类型 (Appointment.type) | 时间段类型 (TimeSlot.appointment_type) | 含义 |
|---------|---------------------------|--------------------------------------|------|
| 卖家流程 | `pick-up` | `pick-up` | 管理员取车（卖家送车） |
| 买家流程 | `pick-up` | `drop-off` | 管理员送车（买家取车） |

**关键理解**：
- `appointment.type` 始终为 `pick-up`（表示这是一个取车预约）
- `time_slot.appointment_type` 决定是谁送车：
  - `pick-up`：管理员取车（卖家流程）
  - `drop-off`：管理员送车（买家流程）

## 关键 API 端点

### 自行车管理
- `POST /bicycles/` - 创建自行车
- `PUT /bicycles/{bike_id}/approve` - 管理员审核通过
- `POST /bicycles/{bike_id}/propose-slots` - 管理员提出时间段
- `POST /bicycles/{bike_id}/confirm` - 管理员确认自行车交易（卖家流程）
- `GET /bicycles/admin/dashboard` - 获取管理后台仪表盘数据

### 时间段管理
- `POST /time_slots/select` - 用户选择时间段
- `PUT /time_slots/confirm/{apt_id}` - 管理员确认时间段
- `PUT /time_slots/confirm-pickup/{apt_id}` - 管理员确认提车（买家流程）

## 状态流转图

### 卖家流程
```
PENDING_APPROVAL → IN_STOCK → PENDING_SELLER_SLOT_SELECTION → 
PENDING_ADMIN_CONFIRMATION_SELLER → RESERVED/SOLD
```

### 买家流程
```
PENDING_APPROVAL → IN_STOCK → PENDING_BUYER_SLOT_SELECTION → 
PENDING_ADMIN_CONFIRMATION_BUYER → PENDING_PICKUP → SOLD
```

## 重要注意事项

1. **两步确认机制**：
   - 卖家流程：卖家选择时间段 → 管理员确认时间段 → 线下交车
   - 买家流程：买家选择时间段 → 管理员确认时间段 → 线下提车 → 管理员确认提车

2. **时间段类型匹配**：
   - 卖家只能选择 `appointment_type = "pick-up"` 的时间段
   - 买家只能选择 `appointment_type = "drop-off"` 的时间段

3. **倒计时机制**：
   - 时间段被选择后，有 24 小时倒计时
   - 超时未确认，时间段自动释放

4. **私信通知**：
   - 管理员提出时间段 → 通知卖家
   - 用户选择时间段 → 通知管理员
   - 管理员确认 → 通知用户
