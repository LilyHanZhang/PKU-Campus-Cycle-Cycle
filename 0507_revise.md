# 管理后台布局改进
现在的管理后台布局太乱了，请按照以下提示进行改进
**不要改变完成的功能，只改进布局**
https://pku-campus-cycle-cycle.vercel.app/admin
## 目前管理后台的布局
卖家线：
1.1. 卖家登记自行车
1.2. 管理员在*待审核车辆*栏目提出时间段。
1.3. 卖家在时间段选择中选择时间
1.4. 管理员在*仪表盘*中点击确认交易，确认卖家审核的时间段
1.5. 线下交易后，管理员在*车辆管理*中点击确认入库
买家线：
2.1 买家锁定坐骑
2.2 管理员在*预约管理*中提出时间段
2.3 买家在时间段选择中选择时间
2.4 管理员在*仪表盘*中点击确认时间段，确认买家审核的时间段
2.5 线下交易后，管理员在*预约管理*中点击确认提车

## 改进的布局
管理后台分为6块：仪表盘；车辆收购；车辆交付；用户管理；车辆管理；公告管理。
每一部分负责任务如下：
1. 仪表盘：对于每一个等待管理员完成的任务，可以提示在仪表盘中，点击能够跳转相应的板块（车辆收购；车辆交付）
2. 车辆收购：对应着卖家线的任务，总共分为三个小栏目：
    - 验车预约：完成本来在1.2的任务，管理员可以提出时间段
    - 收购预约确认：完成本来在1.4的任务，可以确认交易。
    - 库存管理：完成本来在1.5的任务，点击可以确认入库
3. 车辆交付：对应着买家线的任务，总共分为三个小栏目：
    - 提车预约：完成本来在2.2的任务，管理员可以提出时间段
    - 交付预约确认：完成本来在2.4的任务，可以确认买家审核的时间段
    - 交付管理：完成本来在2.5的任务，点击可以确认提车
4. 用户管理：保持原功能
5. 车辆管理：保持原来可以在里面直接操作车辆状态的功能。
6. 公告管理：保持原功能
